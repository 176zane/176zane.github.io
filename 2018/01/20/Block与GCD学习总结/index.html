Block与GCD学习总结

发表于 2018-01-20 | 分类于 iOS 拾遗

Blocks

除了函数中的代码，代码块还包含变量绑定。代码块包含两种类型的绑定：automatic binding 使用的是栈中的内存，而 managed binding 是通过堆创建的。

Blocks 是 C 语言的扩展功能，是带有自动变量（局部变量）的匿名函数。这一概念在其他程序语言中也称为闭包（Closure）、Lambda等。

但是其实在赋值函数指针时，还是用到了函数的名称。
Blocks 还提供了类似 C++ 和 Objective-C 类生成实例或对象来保持变量值的方法，它保持自动变量的值。(避免了声明类的大量代码，也没有使用静态变量、静态全局变量或全局变量时，访问的总是同一个值的问题。)
Block 在被创建时会截获当前的自动变量值，在封闭范围内的自动变量会被保存为常量。要想在 Block 中修改它们的值，需要用 block 声明为可修改的。但是长度可变的数组或包含长度可变数组的结构体是无法声明为 block 的。
如果捕获的自动变量是 Objective-C 对象呢？

是因为 Block 捕获的是指向该对象的指针，而不是对象本身。所以对该对象的操作是可以的，改变 array 这个指针变量却是不可以的。如果要改变需加 __block 修饰符。

Blocks 的语法

与函数指针void(*my_func)(void)类似，Block 有如下特征：1，返回类型可以手动声明也可以由编译器推导；2，具有指定类型的参数列表；3，拥有名称。只要把 * 替换成 ^ 就可以将上面的函数指针转换为一个 Block 的定义了:
int (^blk)(int) = ^int (int count){ return count + 1 };
使用 Block 时通常不需要创建一个 Block 变量，而是在代码中内联它的内容。而且返回值通常可以省略（没有参数时也可忽略参数列表），因此常见的 Block 如下所示：
^(int count){ return count + 1;};
在函数参数或返回值中使用 Block 类型变量时，记述起来很复杂，因此常用 typedef 来解决：
typedef int (^blk_t)(int);

并发性

利用并发性最基础的方法是使用 POSIX 线程来处理程序的不同该部分使其能够独立执行。因为线程是级别较低的 API 你必须手动管理。
mutex 是 mutual exclusion 的缩写，它指的是确保两个线程不会在同一时间进入临界区。
使用performSelectorInBackground:withObject:方法应该注意：1，这些方法运行在各自的线程里，因此你必须为这些 Cocoa 对象创建一个自动释放池，而主自动释放池是与主线程相关的。2，这些方法不能有返回值，而已最多只能有一个参数。方法执行结束后也不会通知你。

GCD

GCD 使用调度队列，只需写下你的代码，把它指派为一个队列，系统就会执行它了。一共有3中类型的队列：

  • 连续队列：每个连续队列都根据指派的顺序执行任务，可以创建任意数量的连续队列，它们会并行操作任务。
  • 并发队列：每个并发队列都能并发执行一个或多个任务，任务会根据指派到队列的顺序开始执行。
  • 主队列： 执行应用程序中主线程的任务。
